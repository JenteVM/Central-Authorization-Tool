# Security Policy

Quick note: As of now I am a 1 man team. I work in my own time and am here just to learn. Do not expect too much support or too frequent updates. I try to at least keep multiple versions up to date but as this is my first big scale project it will probably only support big changes (so full version number like 1.x.x or 2.x.x). This is a project focussed on security and I don't know when something goes at 0.x.0 or at 0.0.x . So sorry if I make mistakes with that.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| 0.1.x   | X |

## Reporting a Vulnerability

I am a one man team and just do this for a hobby. It can't be expected to be updated very frequently, but any update there is should adress all concurrent security issues.
Please make a report for any found security issue not already posted for the version you found the security issue on.
